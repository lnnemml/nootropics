# 0008 — Insert Phase 1.5 (Site Completeness) Before Phase 2

- **Date:** 2026-06-29
- **Status:** Confirmed

## Decision

Before starting Phase 2 (database/product catalog), complete the static
site: product photo + price block, a localStorage-backed cart, About,
FAQ, and legal/technical pages (Terms, Privacy, Refund, Shipping). Full
breakdown: [`../phase-1.5-implementation-plan.md`](../phase-1.5-implementation-plan.md).
This does not renumber Phases 2–7 — it's inserted as "1.5" specifically
to avoid touching the many existing cross-references to those phase
numbers throughout the wiki.

## Context

Anton reviewed the deployed Phase 1 result: the design system holds up
well, but the site is "too generic, not a real e-commerce site yet" —
missing a product photo, price, working cart, and the standard pages
(About, FAQ, legal) any real storefront needs. Building Phase 2's
database layer against an admittedly-incomplete frontend would mean
redoing integration work once the frontend catches up anyway.

## Consequences

- The cart is built localStorage-first, behind a `useCart` hook, so
  Phase 4's database-backed cart is a hook-internals swap, not a UI
  rewrite — see Task 2 in the implementation plan.
- Pricing is explicitly **blocked on Anton supplying real numbers** —
  nothing in the wiki currently has a price. Don't let Claude Code
  invent one to "complete" the task.
- Legal pages are explicitly flagged as AI-drafted templates needing real
  legal review before launch — not a silent assumption, a stated
  constraint (added to root `CLAUDE.md`).

## Revisit if

Phase 1.5 reveals a gap that genuinely needs a new component type or
pattern not in the locked design system (e.g. an FAQ accordion) — that's
a legitimate small addition, not a reason to reopen the whole design
system.
